# AI产品经理模拟面试教练 - 完整开发需求文档

## 项目概述
开发一个AI驱动的产品经理模拟面试Web应用，帮助用户通过实战模拟提升面试能力。用户上传简历（可选粘贴JD），AI扮演资深面试官进行结构化面试，结束后生成个性化复盘报告。

## 核心价值主张
“你的专属AI面试教练，帮你拿下产品经理offer”

## 目标用户画像
- 在校大学生（产品经理方向）准备实习/校招面试
- 转行产品经理的职场新人
- 准备跳槽的初级产品经理

## 核心功能流程

### 1. 用户进入与简历上传
- **首页**：简洁大气的落地页，标题为“AI产品经理模拟面试教练”，副标题“上传简历，开始你的专属面试实战”
- **核心交互**：
  - 简历上传区域（支持PDF格式，拖拽或点击上传）
  - 可选：粘贴企业JD（岗位描述）的文本框，提示“粘贴JD可获得更精准的匹配提问”
  - “开始面试”按钮（上传简历后亮起）
- **技术实现**：上传后后端提取PDF文本，调用LLM解析为结构化JSON（教育背景、实习经历、项目经历、技术技能）

### 2. 模拟面试流程（结构化对话）
采用状态机控制对话流程，每个阶段有独立的Prompt模板：

#### 阶段1：自我介绍
- AI消息：“你好，欢迎参加这次模拟面试。请先用1-2分钟做一个简单的自我介绍，重点突出你的产品经理相关经历和优势。”
- 用户回答后进入下一阶段

#### 阶段2：简历针对性提问（核心）
- AI基于简历中的项目经历和实习经历进行层层递进提问
- 提问逻辑示例：
  - 第一层（背景）：“在你提到的[项目名称]中，你作为产品负责人，当时项目的核心目标和背景是什么？”
  - 第二层（过程）：“你提到做了用户调研，具体用了什么方法？发现了什么关键洞察？这个洞察如何影响了MVP的功能优先级？”
  - 第三层（量化）：“项目上线后，你用哪些指标衡量成功？最终数据表现如何？”
  - 第四层（复盘）：“如果现在让你重做一次，哪个环节你会做得不一样？为什么？”
- 每个项目/实习至少追问2-3轮，模拟真实面试的深度

#### 阶段3：AI技术能力考察
- AI基于用户简历中的技术关键词（如RAG、LLM、Agent、Prompt等）提问
- 预设问题库示例：
  - “你提到用过RAG，能具体讲讲你们当时的检索模块是怎么设计的吗？chunk大小怎么定的？”
  - “你怎么理解AI产品经理和传统产品经理的核心能力差异？”
  - “如果让你为DeepSeek这种大模型设计一个产品功能，你会做什么？为什么？”
  - “在实际项目中，你如何评估一个AI功能的用户体验和商业价值？”
- 如果简历中没有技术关键词，则问通用问题：“你对当前大语言模型的发展趋势有什么看法？这对产品经理意味着什么？”

#### 阶段4：JD匹配提问（如果用户粘贴了JD）
- AI基于JD要求提问，例如：“我看到这个岗位特别强调[JD中的某要求]，请结合你的经历谈谈你如何满足这一点？”

#### 阶段5：结束与反馈
- AI总结：“感谢你的参与，下面我将基于你的回答生成一份复盘报告，帮你梳理优势和待改进点。”
- 触发报告生成

### 3. 面试报告生成
- 面试结束后自动跳转报告页面
- 报告包含：
  - **能力雷达图**（用文字描述，如需求分析:8/10、技术理解:6/10、项目复盘:7/10等）
  - **三个具体亮点**：基于对话中表现好的地方
  - **三个待改进点**：基于对话中暴露的不足
  - **针对性学习建议**：推荐书籍、文章或练习方向
  - **面试完整回顾**：可折叠展示所有QA对
- 报告页面底部：“复制报告链接”按钮，方便用户保存和分享
- 可选：输入邮箱接收报告PDF

## 技术选型建议

### 前端
- 框架：React + TailwindCSS（用v0.dev或Bolt.new生成）
- 核心组件：文件上传、聊天界面、Markdown渲染（用于报告）
- 部署：Vercel（一键部署）

### 后端
- 框架：FastAPI（Python）或 Next.js API Routes
- PDF解析：PyPDF2 或 pdfplumber
- LLM调用：DeepSeek API 或 OpenAI API（初期先用DeepSeek，成本低）
- 数据库：Supabase（存储用户对话记录和报告，有免费额度）
- 部署：Railway 或 Render（与Vercel配合）

### 关键Prompt设计要点
1. **简历解析Prompt**：将PDF文本转为结构化JSON
2. **阶段2提问Prompt**：基于项目经历生成追问，强调层层递进
3. **阶段3提问Prompt**：基于技术关键词抽取问题，如无关键词则问通用技术理解
4. **报告生成Prompt**：基于整个对话分析能力维度，生成亮点、待改进点和建议
